# 目录结构

## 完整文件树

```
packages/agent/
├── package.json                   # 项目配置和依赖
├── tsconfig.json                  # TypeScript 配置
├── tsdown.config.ts               # 构建工具配置（生成 ESM+CJS）
├── vitest.config.ts               # 测试框架配置
│
└── src/                           # 源码目录
    ├── index.ts                   # 公共 API 入口（所有导出）
    ├── agent.ts                   # createUniverseAgent() 主函数
    ├── types.ts                   # TypeScript 类型系统
    ├── config.ts                  # 配置与项目路径管理
    ├── errors.ts                  # ConfigurationError 错误类
    ├── values.ts                  # filesValue 共享状态值
    │
    ├── middleware/                # 中间件层
    │   ├── index.ts              # 中间件统一导出
    │   ├── types.ts              # 中间件专用类型
    │   ├── fs.ts                 # 文件系统中间件（核心）
    │   ├── subagents.ts          # 子代理中间件（同步）
    │   ├── async_subagents.ts    # 子代理中间件（异步/远程）
    │   ├── memory.ts             # AGENTS.md 记忆加载
    │   ├── agent-memory.ts       # 代理记忆（已弃用变体）
    │   ├── skills.ts             # 技能中间件
    │   ├── summarization.ts      # 对话历史摘要压缩
    │   ├── patch_tool_calls.ts   # 修复悬挂工具调用
    │   ├── cache.ts              # 提示缓存断点
    │   ├── completion_callback.ts# 异步子代理完成回调
    │   ├── utils.ts              # 中间件工具函数
    │   └── test.ts               # 测试辅助工具
    │
    ├── backends/                  # 后端实现层
    │   ├── index.ts              # 后端统一导出
    │   ├── protocol.ts           # 协议定义（类型守卫 + 错误类）
    │   ├── state.ts              # StateBackend（存于 LangGraph 状态）
    │   ├── store.ts              # StoreBackend（存于 LangGraph Store）
    │   ├── filesystem.ts         # FilesystemBackend（本地磁盘）
    │   ├── composite.ts          # CompositeBackend（路由组合）
    │   ├── sandbox.ts            # BaseSandbox 抽象基类
    │   ├── local-shell.ts        # LocalShellBackend（本地 Shell）
    │   ├── langsmith.ts          # LangSmithSandbox（云沙箱）
    │   ├── utils.ts              # 后端共享工具函数
    │   ├── v1/
    │   │   └── protocol.ts       # V1 后端协议（已弃用）
    │   └── v2/
    │       └── protocol.ts       # V2 后端协议（当前标准）
    │
    ├── skills/                    # 技能加载器
    │   ├── index.ts              # 导出
    │   └── loader.ts             # 技能文件解析与加载
    │
    └── testing/                   # 测试工具
        └── utils.ts              # 测试辅助函数
```

## 各文件职责详解

### 根目录核心文件

| 文件            | 职责                 | 关键导出                                               |
| --------------- | -------------------- | ------------------------------------------------------ |
| `src/index.ts`  | 公共 API 总入口      | 全部公共类型和函数                                     |
| `src/agent.ts`  | Agent 创建逻辑       | `createUniverseAgent`, `isAnthropicModel`                  |
| `src/types.ts`  | TypeScript 类型定义  | `UniverseAgent`, `CreateUniverseAgentParams`, `AnySubAgent` 等 |
| `src/config.ts` | 项目根检测、路径管理 | `createSettings`, `findProjectRoot`, `Settings`        |
| `src/errors.ts` | 自定义错误类型       | `ConfigurationError`, `ConfigurationErrorCode`         |
| `src/values.ts` | LangGraph 共享状态值 | `filesValue`                                           |

### 中间件目录

| 文件                     | 提供的工具/能力                                                                   | 注册到中间件的钩子                                            |
| ------------------------ | --------------------------------------------------------------------------------- | ------------------------------------------------------------- |
| `fs.ts`                  | `ls`, `read_file`, `write_file`, `edit_file`, `glob`, `grep`, `execute`           | `stateSchema`, `beforeAgent`, `wrapModelCall`, `wrapToolCall` |
| `subagents.ts`           | `task`（子代理调度器）                                                            | `tools`, `wrapModelCall`                                      |
| `async_subagents.ts`     | `create_async_task`, `check_async_task`, `update_async_task`, `cancel_async_task` | `stateSchema`, `tools`                                        |
| `memory.ts`              | 无工具（注入系统提示）                                                            | `wrapModelCall`                                               |
| `skills.ts`              | 无工具（注入系统提示）                                                            | `stateSchema`, `beforeAgent`, `wrapModelCall`                 |
| `summarization.ts`       | 无工具（后台操作）                                                                | `stateSchema`, `beforeAgent`                                  |
| `patch_tool_calls.ts`    | 无工具（消息修复）                                                                | `wrapModelCall`                                               |
| `cache.ts`               | 无工具（缓存控制）                                                                | `wrapModelCall`                                               |
| `completion_callback.ts` | 无工具（回调通知）                                                                | `afterToolCall` 或自定义                                      |

### 后端目录

| 文件             | 后端类型               | 存储位置             | 适用场景                     |
| ---------------- | ---------------------- | -------------------- | ---------------------------- |
| `state.ts`       | `StateBackend`         | LangGraph checkpoint | 默认选项，单线程会话         |
| `store.ts`       | `StoreBackend`         | LangGraph Store      | 跨对话/跨线程持久化          |
| `filesystem.ts`  | `FilesystemBackend`    | 本地磁盘             | 生产环境、需要持久化         |
| `composite.ts`   | `CompositeBackend`     | 多后端路由           | 灵活场景（如部分文件在磁盘） |
| `sandbox.ts`     | `BaseSandbox` (抽象类) | -                    | 扩展基类，实现自定义沙箱     |
| `local-shell.ts` | `LocalShellBackend`    | 本地 Shell           | 本地执行 + 文件系统          |
| `langsmith.ts`   | `LangSmithSandbox`     | LangSmith 云沙箱     | 云端隔离执行环境             |

## 测试文件分布

测试文件与源码同目录，按后缀区分：
- `*.test.ts` — 单元测试（快速，不依赖外部服务）
- `*.int.test.ts` — 集成测试（需要真实 API 或文件系统）

运行命令：
```bash
npm test              # 单元测试
npm run test:int      # 集成测试（需 .env 配置）
npm run test:coverage # 覆盖率报告
```
