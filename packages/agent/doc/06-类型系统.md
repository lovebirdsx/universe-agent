# 类型系统

**文件**：`src/types.ts`

UniverseAgents 的类型系统建立在 LangChain 的类型基础上，通过泛型参数实现完整的类型推断。

---

## 核心类型层级

```
AgentTypeConfig (LangChain)
    └── UniverseAgentTypeConfig     ← 扩展，增加 Subagents 字段

ReactAgent<TTypes> (LangChain)
    └── UniverseAgent<TTypes>       ← 扩展，增加类型品牌

CreateAgentParams (LangChain)
    └── CreateUniverseAgentParams   ← 扩展，增加 backend/memory/skills 等
```

---

## UniverseAgentTypeConfig（类型包）

类型包（Type Bag）模式：将所有泛型参数打包成一个接口，统一传递。

```typescript
interface UniverseAgentTypeConfig<
  TResponse,    // 结构化响应类型（responseFormat 的推断结果）
  TState,       // 自定义状态 Schema 类型
  TContext,     // 上下文 Schema 类型
  TMiddleware,  // 中间件数组类型（用于推断状态）
  TTools,       // 工具数组类型
  TSubagents,   // 子代理数组类型（用于类型安全流式处理）
> extends AgentTypeConfig<TResponse, TState, TContext, TMiddleware, TTools> {
  Subagents: TSubagents;  // 扩展字段：子代理列表
}
```

---

## UniverseAgent 类型

```typescript
type UniverseAgent<TTypes extends UniverseAgentTypeConfig = UniverseAgentTypeConfig>
  = ReactAgent<TTypes> & {
    readonly "~universeAgentTypes": TTypes;  // 类型品牌（phantom type）
  };
```

**类型品牌的作用**：`"~universeAgentTypes"` 是一个不可能被用户代码误操作的 Symbol 字符串，用于从 `UniverseAgent` 实例中提取完整的类型信息，使下游工具类型（如 `InferUniverseAgentSubagents`）能正确工作。

---

## AnySubAgent（子代理联合类型）

```typescript
type AnySubAgent = SubAgent | CompiledSubAgent | AsyncSubAgent;
```

- `SubAgent`：声明式规范，运行时动态创建 Agent
- `CompiledSubAgent`：已编译的 Runnable，直接使用
- `AsyncSubAgent`：远程 Agent Protocol 服务器

---

## SupportedResponseFormat（响应格式）

```typescript
type SupportedResponseFormat = ResponseFormat | TypedToolStrategy<any>;
```

对应三种不同的结构化输出机制：
- `ToolStrategy<T>` — 通过工具调用强制输出 JSON
- `ProviderStrategy<T>` — 通过模型提供商原生格式
- `TypedToolStrategy<T>` — 带类型的工具策略

---

## InferStructuredResponse（响应类型推断）

```typescript
type InferStructuredResponse<T extends SupportedResponseFormat> =
  SupportedResponseFormat extends T
    ? ResponseFormatUndefined          // 未设置 responseFormat → 无结构化响应
    : T extends TypedToolStrategy<infer U> ? U
    : T extends ToolStrategy<infer U> ? U
    : T extends ProviderStrategy<infer U> ? U
    : ResponseFormatUndefined;
```

**使用效果**：
```typescript
// 设置了 responseFormat，structuredResponse 被类型化
const agent = createUniverseAgent({
  responseFormat: ToolStrategy.fromSchema(z.object({ answer: z.string() }))
});
const result = await agent.invoke({ messages: [...] });
result.structuredResponse.answer;  // 类型：string ✓

// 未设置，structuredResponse 不存在
const agent2 = createUniverseAgent();
const result2 = await agent2.invoke({ messages: [...] });
result2.structuredResponse;  // 类型：ResponseFormatUndefined ✓
```

## 子代理中间件类型提取

### ExtractSubAgentMiddleware

从单个子代理定义中提取中间件类型：

```typescript
type ExtractSubAgentMiddleware<T> =
  T extends { middleware?: infer M }
    ? M extends readonly AgentMiddleware[] ? M
    : M extends AgentMiddleware[] ? M
    : readonly []
  : readonly [];
```

### FlattenSubAgentMiddleware

将所有子代理的中间件展平成一个元组：

```typescript
type FlattenSubAgentMiddleware<T extends readonly AnySubAgent[]> =
  T extends readonly [] ? readonly []
  : T extends readonly [infer First, ...infer Rest]
    ? Rest extends readonly AnySubAgent[]
      ? readonly [...ExtractSubAgentMiddleware<First>, ...FlattenSubAgentMiddleware<Rest>]
      : ExtractSubAgentMiddleware<First>
  : readonly [];
```

**用途**：确保子代理自定义中间件中的状态字段也能被主 Agent 的类型系统感知到，实现跨子代理的类型安全状态访问。

## 类型推断工具

### ResolveUniverseAgentTypeConfig

从 UniverseAgent 实例或 UniverseAgentTypeConfig 中解析出类型包：

```typescript
type ResolveUniverseAgentTypeConfig<T> =
  T extends { "~universeAgentTypes": infer Types }
    ? Types extends UniverseAgentTypeConfig ? Types : never
  : T extends UniverseAgentTypeConfig ? T
  : never;
```

### InferUniverseAgentType

从 UniverseAgent 实例中提取任意类型包字段：

```typescript
type InferUniverseAgentType<T, K extends keyof UniverseAgentTypeConfig>
  = ResolveUniverseAgentTypeConfig<T>[K];
```

### InferSubagentByName

按名称提取特定子代理的类型：

```typescript
type InferSubagentByName<T, TName extends string> =
  InferUniverseAgentSubagents<T> extends readonly (infer SA)[]
    ? SA extends { name: TName } ? SA : never
    : never;
```

## 完整类型推断示例

```typescript
import { createUniverseAgent, createMiddleware } from "universe-agent";
import { z } from "zod";

// 定义自定义中间件（含状态）
const ResearchMiddleware = createMiddleware({
  name: "ResearchMiddleware",
  stateSchema: z.object({ researchNotes: z.string().default("") }),
});

// 定义子代理
const researcher = {
  name: "researcher",
  description: "Research assistant",
  systemPrompt: "You are a researcher.",
  middleware: [ResearchMiddleware],
} as const;

// 创建 Agent
const agent = createUniverseAgent({
  middleware: [ResearchMiddleware],
  subagents: [researcher] as const,
  responseFormat: ToolStrategy.fromSchema(z.object({ summary: z.string() })),
});

// 类型推断生效
const result = await agent.invoke({ messages: [...] });
result.researchNotes;           // string ✓（来自 ResearchMiddleware）
result.structuredResponse.summary; // string ✓（来自 responseFormat）

// 子代理类型提取
type SubagentTypes = InferUniverseAgentSubagents<typeof agent>;
type ResearcherType = InferSubagentByName<typeof agent, "researcher">;
```

## MergedUniverseAgentState

```typescript
type MergedUniverseAgentState<
  TMiddleware extends readonly AgentMiddleware[],
  TSubagents extends readonly AnySubAgent[],
> = InferMiddlewareStates<TMiddleware> & InferSubAgentMiddlewareStates<TSubagents>;
```

将主 Agent 中间件状态与所有子代理中间件状态合并为一个联合类型，用于完整的状态类型描述。
