# 摘要中间件

**文件**：`src/middleware/summarization.ts`

摘要中间件在对话历史超过 token 阈值时自动压缩历史记录，防止上下文窗口溢出，同时将摘要持久化到后端文件系统。

---

## 工作原理

```
检测到对话历史过长
    │
    ▼
调用 LLM 生成摘要
（对历史消息进行压缩，保留关键信息）
    │
    ▼
持久化摘要到文件
/conversation_history/{thread_id}.md
    │
    ▼
截断历史消息
（保留最近 N 条消息 + 摘要作为新的"基础"）
    │
    ▼
在系统提示中注入摘要引用
（告知 LLM 参考历史摘要）
```

---

## 触发策略（ContextSize）

```typescript
type ContextSize = {
  type: "messages" | "tokens" | "fraction";
  value: number;
}
```

| 类型 | 说明 | 示例 |
|------|------|------|
| `messages` | 消息数量阈值 | `{ type: "messages", value: 50 }` |
| `tokens` | token 数量阈值 | `{ type: "tokens", value: 100000 }` |
| `fraction` | 上下文窗口的比例 | `{ type: "fraction", value: 0.8 }` |

---

## 默认配置（computeSummarizationDefaults）

根据模型自动计算最优参数：

```typescript
function computeSummarizationDefaults(model) {
  // 从模型名称/配置中提取上下文窗口大小
  const contextWindow = getModelContextWindow(model);

  return {
    triggerThreshold: {
      type: "tokens",
      value: contextWindow * 0.7,  // 70% 时触发
    },
    keepLastN: 10,  // 触发后保留最近 10 条消息
  };
}
```

---

## 摘要持久化

摘要内容写入后端文件系统：

```
/conversation_history/{thread_id}.md
```

格式：
```markdown
# Conversation Summary

## Summary (as of {timestamp})

{LLM 生成的摘要内容}

---
*Summarized from {n} messages at token count {count}*
```

## 状态 Schema

```typescript
stateSchema: z.object({
  summarization: z.object({
    lastSummarizedAt: z.number().optional(),   // 上次摘要的 token 数
    summaryPath: z.string().optional(),         // 摘要文件路径
    summaryCount: z.number().default(0),       // 已执行摘要次数
  }).default({})
})
```

---

## 工具参数截断

对于较旧的消息，摘要中间件会截断工具调用的参数（减少 token 占用）：

```typescript
// 旧消息的工具调用参数被截断为固定提示
toolCall.args = { __truncated__: "Tool args truncated to save tokens" };
```

这样保留了工具调用历史（LLM 知道调用过什么工具），但减少了存储的 token。

## 选项参数

```typescript
interface SummarizationMiddlewareOptions {
  backend?: AnyBackendProtocol | BackendFactory;  // 用于持久化摘要
  model?: string | BaseLanguageModel;             // 用于自动计算阈值
  triggerThreshold?: ContextSize;                 // 手动设置触发阈值
  keepLastN?: number;                             // 触发后保留消息数
  summarizationModel?: string | BaseLanguageModel; // 专用摘要模型（可以不同于主模型）
}
```

## beforeAgent 钩子实现

```typescript
async beforeAgent(state) {
  // 1. 检查是否达到触发阈值
  const tokenCount = estimateTokens(state.messages);
  if (tokenCount < triggerThreshold.value) return;

  // 2. 调用 LLM 生成摘要
  const summary = await summarizationModel.invoke([
    new SystemMessage("Summarize the following conversation history..."),
    ...state.messages,
  ]);

  // 3. 持久化摘要
  const path = `/conversation_history/${threadId}.md`;
  await resolvedBackend.write(path, formatSummary(summary));

  // 4. 更新状态（截断历史 + 更新摘要记录）
  return {
    messages: keepLastN(state.messages),
    summarization: { lastSummarizedAt: tokenCount, summaryPath: path }
  };
}
```
