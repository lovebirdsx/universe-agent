# 后端协议

**文件**：`src/backends/protocol.ts`、`src/backends/v2/protocol.ts`

后端协议定义了所有文件操作的抽象接口，使中间件代码与具体存储实现解耦。

---

## BackendProtocolV2（当前标准）

```typescript
interface BackendProtocolV2 {
  // 列出目录内容
  ls(path: string): MaybePromise<LsResult>;

  // 读取文件（支持分页）
  read(filePath: string, offset?: number, limit?: number): MaybePromise<ReadResult>;

  // 读取原始文件数据（不分页）
  readRaw(filePath: string): MaybePromise<ReadRawResult>;

  // 写入文件（覆盖创建）
  write(filePath: string, content: string): MaybePromise<WriteResult>;

  // 精确字符串替换
  edit(
    filePath: string,
    oldString: string,
    newString: string,
    replaceAll?: boolean
  ): MaybePromise<EditResult>;

  // Glob 模式匹配
  glob(pattern: string, path?: string): MaybePromise<GlobResult>;

  // 文本内容搜索
  grep(pattern: string, path?: string, glob?: string | null): MaybePromise<GrepResult>;
}
```

## SandboxBackendProtocolV2（含执行能力）

```typescript
interface SandboxBackendProtocolV2 extends BackendProtocolV2 {
  // Shell 命令执行
  execute(command: string): MaybePromise<ExecuteResponse>;

  // 沙箱标识符（非空字符串）
  id: string;

  // 可选：沙箱生命周期管理
  list?(options?: SandboxListOptions): MaybePromise<SandboxListResponse>;
  getOrCreate?(options?: SandboxGetOrCreateOptions): MaybePromise<SandboxInfo>;
  delete?(options: SandboxDeleteOptions): MaybePromise<void>;

  // 可选：文件批量传输
  downloadFiles?(filePaths: string[]): MaybePromise<FileDownloadResponse[]>;
  uploadFiles?(files: Array<{ path: string; content: Uint8Array }>): MaybePromise<FileUploadResponse[]>;
}
```

## 操作结果类型

所有操作都返回包含 `error` 字段的结构体（而非抛出异常）：

### ReadResult

```typescript
interface ReadResult {
  error?: string;                  // 错误消息（失败时）
  content?: string | Uint8Array;   // 文件内容（成功时）
  mimeType?: string;               // MIME 类型
}
```

### WriteResult

```typescript
interface WriteResult {
  error?: string;
  path?: string;
  filesUpdate?: Record<string, FileData> | null;  // StateBackend 使用
  metadata?: Record<string, unknown>;
}
```

`filesUpdate` 字段的语义：
- 非 null（如 `{ "/path/to/file.txt": FileData }` ）：StateBackend 使用，需要通过 LangGraph `Command` 更新状态
- `null`：外部存储（FilesystemBackend 等），已直接写入，不需要状态更新

### EditResult

```typescript
interface EditResult {
  error?: string;
  path?: string;
  filesUpdate?: Record<string, FileData> | null;
  occurrences?: number;   // 实际替换次数
  metadata?: Record<string, unknown>;
}
```

### ExecuteResponse

```typescript
interface ExecuteResponse {
  output: string;           // stdout + stderr 合并输出
  exitCode: number | null;  // 退出码（null 表示未获取到）
  truncated: boolean;       // 输出是否被截断
}
```

### LsResult

```typescript
interface LsResult {
  error?: string;
  files?: FileInfo[];
}

interface FileInfo {
  path: string;
  is_dir?: boolean;
  size?: number;
  modified_at?: string;
}
```

### GrepResult

```typescript
interface GrepResult {
  error?: string;
  matches?: GrepMatch[];
}

interface GrepMatch {
  path: string;
  line: number;    // 1-indexed
  text: string;
}
```

function isSandboxBackend(backend: unknown): backend is SandboxBackendProtocolV2 {
  return (
    backend != null &&
    typeof backend === "object" &&
    typeof (backend as any).execute === "function" &&
    typeof (backend as any).id === "string" &&
    (backend as any).id !== ""
  );

## resolveBackend（版本适配）

```typescript
async function resolveBackend(
  backend: AnyBackendProtocol | BackendFactory,
  runtime: BackendRuntime | ToolRuntime,
): Promise<BackendProtocolV2> {
  if (typeof backend === "function") {
    // BackendFactory：调用工厂函数
    const resolved = await backend(runtime as BackendRuntime);
    return isSandboxProtocol(resolved)
      ? adaptSandboxProtocol(resolved)   // V1 沙箱 → V2 适配
      : adaptBackendProtocol(resolved);  // V1 普通 → V2 适配
  }
  // 直接实例：适配到 V2
  return isSandboxProtocol(backend)
    ? adaptSandboxProtocol(backend)
    : adaptBackendProtocol(backend);
}
```

**向后兼容**：`adaptBackendProtocol` 和 `adaptSandboxProtocol` 将 V1 接口适配到 V2，确保旧实现仍可使用。
