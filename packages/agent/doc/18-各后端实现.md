## StateBackend

**文件**：`src/backends/state.ts`

### 定位

最简单的后端，将文件存储在 LangGraph 的检查点状态（`files` 字段）中。零配置，开箱即用。

### 特点

- **存储位置**：LangGraph checkpoint（内存或持久化，取决于 checkpointer 配置）
- **作用域**：单个对话线程内共享
- **持久化**：仅在 checkpointer 存在时持久化
- **跨线程**：不支持（每个线程独立）

### 实现要点

```typescript
class StateBackend implements BackendProtocolV2 {
  // 通过 getCurrentTaskInput() 从 LangGraph 上下文读取状态
  // 通过 __pregel_send 发送状态更新
  
  async read(filePath, offset?, limit?) {
    const state = getCurrentTaskInput();
    const files = state.files as FilesRecord;
    const fileData = files[filePath];
    
    if (!fileData) return { error: `File not found: ${filePath}` };
    
    // 处理 V1/V2 格式差异
    const content = isFileDataV1(fileData)
      ? fileData.content.join("\n")
      : fileData.content;
    
    // 应用分页
    const lines = content.split("\n");
    const pageContent = lines.slice(offset, offset + limit).join("\n");
    return { content: pageContent, mimeType: "text/plain" };
  }

  async write(filePath, content) {
    const now = new Date().toISOString();
    const fileData: FileDataV2 = {
      content,
      mimeType: "text/plain",
      created_at: now,
      modified_at: now,
    };

    // 通过 Command 更新 LangGraph 状态
    const filesUpdate = { [filePath]: fileData };
    return { path: filePath, filesUpdate };
  }
}
```

### 写入流程

StateBackend 的 `write` 方法返回 `filesUpdate`，调用方（`write_file` 工具）负责将其包装成 `Command` 发送给 LangGraph：

```typescript
// write_file 工具中
const result = await resolvedBackend.write(file_path, content);
if (result.filesUpdate) {
  return new Command({
    update: { files: result.filesUpdate, messages: [message] },
  });
}
```

## StoreBackend

**文件**：`src/backends/store.ts`

### 定位

使用 LangGraph Store（持久化存储）保存文件，支持跨线程、跨会话访问。

### 特点

- **存储位置**：LangGraph BaseStore（数据库）
- **作用域**：按命名空间隔离，可跨线程共享
- **持久化**：是（数据库级别）
- **跨线程**：支持（共享命名空间）

### 命名空间

```typescript
interface StoreBackendOptions {
  namespace?: string[];  // 默认 ["files"]
  // 或自定义工厂函数
  namespaceFactory?: StoreBackendNamespaceFactory;
}

type StoreBackendNamespaceFactory = (context: StoreBackendContext) => string[];

interface StoreBackendContext {
  threadId?: string;
  userId?: string;
  assistantId?: string;
}
```

示例：按用户隔离文件
```typescript
backend: new StoreBackend({
  namespaceFactory: ({ userId }) => ["files", userId ?? "anonymous"]
})
```

### 实现要点

```typescript
class StoreBackend implements BackendProtocolV2 {
  async write(filePath, content) {
    const store = getStore();  // LangGraph 提供的 Store 实例
    const key = pathToKey(filePath);  // 路径转换为 Store key
    
    await store.put(this.namespace, key, {
      content,
      mimeType: "text/plain",
      created_at: now,
      modified_at: now,
    });

    return { path: filePath, filesUpdate: null };  // null：已直接写入 Store
  }
}
```

## FilesystemBackend

**文件**：`src/backends/filesystem.ts`

### 定位

直接操作本地文件系统，适合需要真正持久化文件的生产场景。

### 特点

- **存储位置**：本地磁盘（rootDir 参数指定）
- **路径安全**：防止路径遍历（`..` 攻击）
- **符号链接保护**：使用 `O_NOFOLLOW` 标志
- **虚拟模式**：可选的沙箱模式（路径隔离）

### 构造参数

```typescript
interface FilesystemBackendOptions {
  rootDir?: string;       // 根目录（默认：process.cwd()）
  virtualMode?: boolean;  // 虚拟模式（路径不出根目录）
  maxFileSizeMb?: number; // 最大文件大小（默认 10MB）
}
```

### 安全措施

```typescript
// 路径解析安全检查
function resolveSafePath(rootDir: string, filePath: string): string {
  const resolved = path.resolve(rootDir, filePath.replace(/^\//, ""));
  
  // 确保解析后的路径在 rootDir 内
  if (!resolved.startsWith(rootDir)) {
    throw new Error(`Path traversal detected: ${filePath}`);
  }
  
  return resolved;
}
```

### 实现要点

```typescript
class FilesystemBackend implements BackendProtocolV2 {
  async write(filePath, content) {
    const safePath = resolveSafePath(this.rootDir, filePath);
    await fs.mkdir(path.dirname(safePath), { recursive: true });
    await fs.writeFile(safePath, content, "utf-8");
    
    return {
      path: filePath,
      filesUpdate: null,  // null：直接写入磁盘，不需要状态更新
    };
  }

  async grep(pattern, path = "/", glob = null) {
    // 使用 ripgrep 或内置搜索
    const files = await fastGlob(glob ? `**/${glob}` : "**/*", { cwd: rootPath });
    const matches: GrepMatch[] = [];

    for (const file of files) {
      const content = await fs.readFile(file, "utf-8");
      const lines = content.split("\n");
      for (let i = 0; i < lines.length; i++) {
        if (lines[i].includes(pattern)) {
          matches.push({ path: file, line: i + 1, text: lines[i] });
        }
      }
    }

    return { matches };
  }
}
```

## CompositeBackend

**文件**：`src/backends/composite.ts`

### 定位

将多个后端组合，按路径前缀路由到不同后端。

### 使用场景

- 大文件存储在文件系统，小文件/临时文件存储在状态
- 某些路径存储在 Store（持久化），其他路径存储在状态

### 配置示例

```typescript
const backend = new CompositeBackend([
  {
    prefix: "/projects/",         // 匹配 /projects/ 开头的路径
    backend: new FilesystemBackend({ rootDir: "/data/projects" }),
  },
  {
    prefix: "/tmp/",              // 匹配 /tmp/ 开头的路径
    backend: new StateBackend(),
  },
  {
    prefix: "/",                  // 默认（兜底）
    backend: new StoreBackend(),
  },
]);
```

### 路由逻辑

```typescript
class CompositeBackend implements BackendProtocolV2 {
  private findBackend(filePath: string): BackendProtocolV2 {
    // 按注册顺序查找第一个匹配的前缀
    for (const { prefix, backend } of this.routes) {
      if (filePath.startsWith(prefix)) return backend;
    }
    throw new Error(`No backend found for path: ${filePath}`);
  }

  async write(filePath, content) {
    return this.findBackend(filePath).write(filePath, content);
  }

  async ls(path) {
    // ls 需要聚合多个后端的结果
    const backends = this.getBackendsForPath(path);
    const results = await Promise.all(backends.map(b => b.ls(path)));
    return { files: results.flatMap(r => r.files ?? []) };
  }
}
```

## 后端选择指南

| 场景            | 推荐后端                               |
| --------------- | -------------------------------------- |
| 快速原型、测试  | `StateBackend`（默认，零配置）         |
| 跨会话持久化    | `StoreBackend`（配合 LangGraph Store） |
| 真正的文件操作  | `FilesystemBackend`                    |
| 混合场景        | `CompositeBackend`                     |
| 云端安全执行    | `LangSmithSandbox`                     |
| 本地 Shell 执行 | `LocalShellBackend`                    |
