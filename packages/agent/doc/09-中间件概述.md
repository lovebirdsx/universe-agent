# 中间件概述

**目录**：`src/middleware/`

中间件是 DeepAgents 功能扩展的核心机制，基于 LangChain 的 `createMiddleware` API 实现。

---

## 中间件机制

中间件通过 `langchain.createMiddleware()` 创建，可以：

1. **注册状态 Schema**（`stateSchema`）：向 LangGraph 状态添加新字段
2. **注册工具**（`tools`）：向 Agent 添加新的可调用工具
3. **拦截 Agent 执行**（`beforeAgent`）：在 LLM 推理前执行逻辑
4. **拦截模型调用**（`wrapModelCall`）：修改发给 LLM 的请求（系统提示、工具列表等）
5. **拦截工具调用**（`wrapToolCall`）：修改工具执行前后的行为

```typescript
createMiddleware({
  name: "MyMiddleware",
  stateSchema: z.object({ myField: z.string().default("") }),
  tools: [myTool],
  beforeAgent: async (state) => { ... },           // 返回状态更新或 undefined
  wrapModelCall: async (request, handler) => { ... }, // 返回修改后的 handler 结果
  wrapToolCall: async (request, handler) => { ... },  // 返回工具结果
})
```

## 执行顺序与生命周期

```
agent.invoke() 调用
    │
    ▼ ─────────────── beforeAgent 阶段 ─────────────────────
    │
    │  middleware[1].beforeAgent(state)    → 状态更新（可选）
    │  middleware[2].beforeAgent(state)    → 状态更新（可选）
    │  ...
    │
    ▼ ─────────────── LLM 推理循环 ────────────────────────
    │
    │  ┌─ wrapModelCall 阶段 ─────────────────────────────┐
    │  │  middleware[n].wrapModelCall(request, handler)    │
    │  │  middleware[n-1].wrapModelCall(request, handler)  │
    │  │  ...（洋葱模型，最后注册的最先包裹）               │
    │  │  → LLM 实际调用                                   │
    │  └───────────────────────────────────────────────────┘
    │
    │  LLM 返回工具调用决策
    │
    │  ┌─ wrapToolCall 阶段 ─────────────────────────────┐
    │  │  middleware[n].wrapToolCall(request, handler)   │
    │  │  ...                                            │
    │  │  → 实际工具执行                                  │
    │  └──────────────────────────────────────────────────┘
    │
    │  循环直到 LLM 不再调用工具
    │
    ▼ ─────────────── 结束 ─────────────────────────────────
```

**注意**：`wrapModelCall` 和 `wrapToolCall` 使用**洋葱模型**（Onion Model），即中间件注册顺序的反向包裹：最后注册的中间件最先获得控制权。

### 固定注册（始终包含）

| 序号 | 中间件                           | 文件                  | 提供能力                            |
| ---- | -------------------------------- | --------------------- | ----------------------------------- |
| 1    | `todoListMiddleware`             | langchain 内置        | `write_todos` 工具（TODO 列表管理） |
| 2    | `createFilesystemMiddleware`     | `fs.ts`               | 6 个文件系统工具 + 大结果驱逐       |
| 3    | `createSubAgentMiddleware`       | `subagents.ts`        | `task` 工具（子代理委派）           |
| 4    | `createSummarizationMiddleware`  | `summarization.ts`    | 对话历史自动压缩                    |
| 5    | `createPatchToolCallsMiddleware` | `patch_tool_calls.ts` | 悬挂工具调用修复                    |

### 条件注册

| 条件                  | 中间件                             | 文件                 | 提供能力               |
| --------------------- | ---------------------------------- | -------------------- | ---------------------- |
| `skills` 参数非空     | `createSkillsMiddleware`           | `skills.ts`          | 技能加载与渐进式披露   |
| `asyncSubAgents` 非空 | `createAsyncSubAgentMiddleware`    | `async_subagents.ts` | 4 个异步任务工具       |

| `interruptOn` 非空    | `humanInTheLoopMiddleware`         | langchain 内置       | 工具执行前人工审批     |

## 中间件状态 Schema 汇总

每个注册状态 Schema 的中间件都会向 LangGraph 状态添加字段：

| 字段名           | 类型                       | 来源中间件                | 说明                     |
| ---------------- | -------------------------- | ------------------------- | ------------------------ |
| `files`          | `Record<string, FileData>` | `FilesystemMiddleware`    | 文件内容（Reducer 合并） |
| `asyncTasks`     | `AsyncTask[]`              | `AsyncSubAgentMiddleware` | 异步任务状态列表         |
| `skillsMetadata` | `SkillMetadata[]`          | `SkillsMiddleware`        | 已加载的技能元数据       |
| `memoryContents` | `Record<string, string>`   | `MemoryMiddleware`        | 已加载的记忆内容         |
| `summarization`  | `SummarizationState`       | `SummarizationMiddleware` | 摘要历史和统计           |

## 中间件传参模式

DeepAgents 的中间件工厂函数统一接受一个 `options` 对象：

```typescript
// 标准模式
const middleware = createXxxMiddleware({
  backend,      // 文件后端（几乎所有中间件都需要）
  model,        // 模型（摘要中间件需要）
  sources,      // 文件路径列表（技能、记忆中间件需要）
  // ...其他选项
});
```

**后端统一注入**：几乎所有中间件都通过 `backend` 参数接受同一个后端实例，实现后端替换时只需在 `createDeepAgent` 传一次。

## 自定义中间件示例

```typescript
import { createMiddleware, tool } from "langchain";
import { z } from "zod";

// 最简中间件（只添加工具，不添加状态）
const SimpleToolMiddleware = createMiddleware({
  name: "SimpleToolMiddleware",
  tools: [
    tool(
      async (input) => `Hello, ${input.name}!`,
      {
        name: "greet",
        description: "Greet someone",
        schema: z.object({ name: z.string() }),
      }
    )
  ],
});

// 带状态的中间件
const CounterMiddleware = createMiddleware({
  name: "CounterMiddleware",
  stateSchema: z.object({ count: z.number().default(0) }),
  wrapToolCall: async (request, handler) => {
    const result = await handler(request);
    // 注意：实际更新状态需要通过 Command
    return result;
  },
});

const agent = createDeepAgent({
  middleware: [SimpleToolMiddleware, CounterMiddleware],
});
```
