# 异步子代理中间件

**文件**：`src/middleware/async_subagents.ts`

异步子代理中间件支持将任务委派给远程 LangGraph 服务器（Agent Protocol），实现长时间运行的异步任务管理。

---

## 核心概念

与同步子代理（`task` 工具）不同，异步子代理：
- **不阻塞**：任务发起后主代理继续执行
- **远程执行**：通过 HTTP/Agent Protocol 连接远程 LangGraph 服务器
- **轮询状态**：通过 `check_async_task` 工具查询任务进度

---

## AsyncSubAgent 规范

```typescript
interface AsyncSubAgent {
  name: string;           // 子代理标识符
  description: string;    // 用途说明（展示给 LLM）
  graphId: string;        // 远程 LangGraph 图 ID
  url?: string;          // Agent Protocol 服务器 URL
  headers?: Record<string, string>;  // 自定义请求头（如 Auth Token）
}
```

**识别方式**：通过 `graphId` 字段判断是否为异步子代理：

```typescript
export function isAsyncSubAgent(agent: AnySubAgent): agent is AsyncSubAgent {
  return "graphId" in agent;
}
```

## AsyncTask 状态

```typescript
interface AsyncTask {
  taskId: string;        // 唯一任务 ID（本地生成）
  agentName: string;     // 对应的异步子代理名称
  threadId: string;      // 远程 LangGraph 线程 ID
  runId: string;         // 运行 ID
  status: AsyncTaskStatus; // 当前状态
  createdAt: string;     // ISO 时间戳
  description?: string;  // 任务描述（便于 LLM 追踪）
}

type AsyncTaskStatus =
  | "pending"    // 等待执行
  | "running"    // 正在执行
  | "success"    // 执行成功
  | "error"      // 执行失败
  | "cancelled"; // 已取消
```

## 提供的工具

### create_async_task

```typescript
schema: z.object({
  agentName: z.string(),      // 使用哪个异步子代理
  description: z.string(),    // 任务描述（发给远程 Agent 的消息）
})
```

执行流程：
1. 通过 LangGraph SDK 创建远程线程
2. 在线程上启动运行（`invoke`/`stream`）
3. 记录 `threadId` 和 `runId` 到本地状态
4. 返回任务 ID，供后续轮询使用

### check_async_task

```typescript
schema: z.object({
  taskId: z.string(),         // 要查询的任务 ID
})
```

执行流程：
1. 从本地状态找到对应任务
2. 通过 LangGraph SDK 查询远程运行状态
3. 更新本地状态（`asyncTasks`）
4. 返回当前状态和结果（如果已完成）

### update_async_task

```typescript
schema: z.object({
  taskId: z.string(),
  message: z.string(),        // 发送给运行中子代理的追加消息
})
```

允许主代理向正在运行的子代理发送追加指令。

### cancel_async_task

```typescript
schema: z.object({
  taskId: z.string(),
})
```

取消一个正在运行的异步任务。

## 与 LangGraph SDK 的集成

```typescript
import { Client } from "@langchain/langgraph-sdk";

// 创建任务时
const client = new Client({ apiUrl: asyncSubAgent.url, headers });
const thread = await client.threads.create();
const run = await client.runs.create(thread.thread_id, asyncSubAgent.graphId, {
  input: { messages: [{ role: "user", content: description }] },
});
```

## 使用示例

```typescript
const agent = createUniverseAgent({
  subagents: [
    {
      name: "data-processor",
      description: "Long-running data processing agent",
      graphId: "data-processor-graph",
      url: "https://my-langgraph-server.com",
      headers: { "X-Api-Key": process.env.API_KEY },
    }
  ]
});

// LLM 会使用工具：
// 1. create_async_task → 启动远程任务，得到 taskId
// 2. 继续处理其他工作...
// 3. check_async_task → 轮询任务完成状态
// 4. 获取结果，继续执行
```
