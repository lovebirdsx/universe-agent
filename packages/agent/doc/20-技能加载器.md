# 技能加载器

**文件**：`src/skills/loader.ts`、`src/skills/index.ts`

技能加载器提供独立于中间件的技能文件解析工具，可在不启动 Agent 的情况下枚举和解析技能。

---

## parseSkillMetadata

解析 SKILL.md 文件的 YAML frontmatter：

```typescript
function parseSkillMetadata(
  content: string,
  filePath: string,
): SkillMetadata | null
```

### 解析流程

```
1. 查找 frontmatter（--- ... --- 格式）
   ├─ 未找到 → 返回 null
   └─ 找到 → 继续

2. 解析 YAML
   ├─ 解析失败 → 返回 null
   └─ 解析成功 → 继续

3. 验证必填字段
   ├─ 缺少 name → 返回 null
   ├─ 缺少 description → 返回 null
   └─ 验证通过 → 继续

4. 验证约束
   ├─ name > 64 字符 → 返回 null
   ├─ description > 1024 字符 → 返回 null
   ├─ name 不符合 /^[a-z0-9-]+$/ → 返回 null
   └─ 验证通过 → 返回 SkillMetadata
```

### 名称格式约束

技能名只允许：
- 小写字母（`a-z`）
- 数字（`0-9`）
- 连字符（`-`）

不允许：下划线、大写字母、空格等

## listSkills

从文件系统目录枚举技能：

```typescript
async function listSkills(options: ListSkillsOptions): Promise<SkillMetadata[]>

interface ListSkillsOptions {
  rootDir?: string;     // 搜索根目录（默认 process.cwd()）
  sources?: string[];   // 相对于 rootDir 的技能目录列表
}
```

### 搜索逻辑

```typescript
async function listSkills({ rootDir, sources }) {
  const skills: SkillMetadata[] = [];

  for (const source of sources) {
    const sourcePath = path.join(rootDir, source);

    // 使用 fast-glob 查找所有 SKILL.md 文件
    const skillFiles = await fastGlob("**/SKILL.md", {
      cwd: sourcePath,
      absolute: true,
    });

    for (const skillFile of skillFiles) {
      const content = await fs.readFile(skillFile, "utf-8");

      // 大小检查
      if (Buffer.byteLength(content) > MAX_SKILL_FILE_SIZE) continue;

      const metadata = parseSkillMetadata(content, skillFile);
      if (metadata) skills.push(metadata);
    }
  }

  return skills;
}
```

## 技能文件的完整内容

技能文件包含两部分：
1. **YAML frontmatter**（元数据，必须）
2. **Markdown 正文**（技能指令，注入到系统提示）

```markdown
---
name: code-review
description: Systematically review code for bugs, security issues, and best practices
license: MIT
allowedTools:
  - read_file
  - grep
  - glob
---

# Code Review Skill

When asked to review code, follow this systematic process:

## Step 1: Understand Context
- Read the README or documentation first
- Understand the project structure

## Step 2: Review the Code
...（具体技能指令）
```

中间件只在元数据阶段加载 frontmatter，完整 Markdown 正文按需加载注入系统提示。

## SkillMetadata 完整类型

```typescript
interface SkillMetadata {
  name: string;                          // 技能唯一名称
  description: string;                   // 技能描述（展示给 LLM）
  path: string;                          // SKILL.md 文件在后端中的绝对路径
  license?: string | null;               // 开源许可证（如 "MIT"）
  compatibility?: string | null;         // 环境要求说明（如 "需要 Node.js 18+"）
  metadata?: Record<string, string>;     // 自定义键值对（版本、作者等）
  allowedTools?: string[];               // 技能建议使用的工具名列表
}
```

## 使用示例

```typescript
import { listSkills, parseSkillMetadata } from "universe-agent";

// 列出目录中的所有技能
const skills = await listSkills({
  rootDir: "/home/user/.universe-agent",
  sources: ["skills/"],
});

console.log(skills);
// [
//   {
//     name: "web-search",
//     description: "Search the web using Tavily",
//     path: "/home/user/.universe-agent/skills/web-search/SKILL.md",
//     allowedTools: ["search_web"],
//   },
//   ...
// ]

// 解析单个技能文件
const content = fs.readFileSync("/path/to/SKILL.md", "utf-8");
const metadata = parseSkillMetadata(content, "/path/to/SKILL.md");
```
