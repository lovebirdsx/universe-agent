## patch_tool_calls.ts — 工具调用修复中间件

**文件**：`src/middleware/patch_tool_calls.ts`

### 问题背景

某些情况下会产生"悬挂工具调用"：
- LLM 的消息中包含 `tool_use` block（AI 决定调用工具）
- 但对应的 `ToolMessage`（工具执行结果）丢失了
- 这会导致后续请求因消息历史不完整而失败

### 解决方案

在 `wrapModelCall` 中扫描并修复消息历史：

```typescript
// wrapModelCall 中的修复逻辑
function patchToolCalls(messages) {
  const result = [];
  for (const msg of messages) {
    result.push(msg);

    // 检查 AI 消息是否有未回应的工具调用
    if (isAIMessage(msg) && msg.tool_calls?.length > 0) {
      const pendingIds = msg.tool_calls.map(tc => tc.id);

      // 检查下一条消息是否有对应的 ToolMessage
      // 如果没有，插入占位 ToolMessage
      for (const id of pendingIds) {
        if (!hasMatchingToolMessage(messages, id)) {
          result.push(new ToolMessage({
            content: "Tool call was interrupted",
            tool_call_id: id,
            name: msg.tool_calls.find(tc => tc.id === id)?.name ?? "unknown",
          }));
        }
      }
    }
  }
  return result;
}
```

### 触发场景

- Agent 在工具执行中途被中断（如 SIGINT）
- 人工审批中断后恢复时状态不一致
- 跨会话恢复时历史消息缺失

## cache.ts — 缓存断点中间件

**文件**：`src/middleware/cache.ts`

### 功能

为 Anthropic 模型的系统提示添加精确的 `cache_control` 断点，控制哪部分内容被缓存：

```typescript
export function createCacheBreakpointMiddleware() {
  return createMiddleware({
    name: "CacheBreakpointMiddleware",
    wrapModelCall: async (request, handler) => {
      // 在系统消息的最后一个 block 添加缓存断点
      const systemMessage = addCacheBreakpoint(request.systemMessage);
      return handler({ ...request, systemMessage });
    },
  });
}
```

### 缓存策略

```
系统提示结构：
  [BASE_AGENT_PROMPT]           ← 固定，缓存整个生命周期
  [FILESYSTEM_PROMPT]           ← 固定，缓存
  [SKILLS_PROMPT]               ← 相对固定，缓存
  [MEMORY_CONTENT]              ← 相对固定，缓存（addCacheControl=true）
  ← 缓存断点（cache_control: ephemeral）
  [TASK_SYSTEM_PROMPT]          ← 动态部分，不缓存
```

与 `anthropicPromptCachingMiddleware` 配合：
- 后者负责将 `cache_control` 块传递给 Anthropic API
- 前者负责决定在哪里设置断点

## completion_callback.ts — 完成回调中间件

**文件**：`src/middleware/completion_callback.ts`

### 功能

为异步子代理提供完成通知机制。当远程 Agent 完成时，通过回调通知主 Agent：

```typescript
interface CompletionCallbackOptions {
  onComplete: (taskId: string, result: unknown) => void | Promise<void>;
  onError?: (taskId: string, error: Error) => void | Promise<void>;
}

export function createCompletionCallbackMiddleware(
  options: CompletionCallbackOptions
) {
  return createMiddleware({
    name: "CompletionCallbackMiddleware",
    // 在工具调用后检查异步任务是否完成
    wrapToolCall: async (request, handler) => {
      const result = await handler(request);
      // 检查并触发回调...
      return result;
    },
  });
}
```
