# 子代理中间件

**文件**：`src/middleware/subagents.ts`

子代理中间件为主 Agent 提供 `task` 工具，使其能够将复杂子任务委派给专用子代理并行处理。

---

## 核心概念

### SubAgent（声明式规范）

```typescript
interface SubAgent {
  name: string;                    // 标识符，task 工具中的 subagent_type
  description: string;             // 展示给 LLM 用于选择
  systemPrompt: string;            // 子代理的系统提示
  tools?: StructuredTool[];        // 工具列表（默认继承主 Agent 工具）
  model?: LanguageModelLike | string; // 模型（默认继承主 Agent 模型）
  middleware?: readonly AgentMiddleware[];  // 附加中间件
  interruptOn?: Record<string, InterruptOnConfig>; // 人工审批
  skills?: string[];               // 技能路径
  responseFormat?: CreateAgentParams["responseFormat"]; // 结构化输出
}
```

### CompiledSubAgent（已编译代理）

```typescript
interface CompiledSubAgent<TRunnable extends ReactAgent<any> | Runnable> {
  name: string;
  description: string;
  runnable: TRunnable;  // 已经编译好的 Agent 实例，直接使用
}
```

**区别**：`SubAgent` 在运行时由中间件动态编译，`CompiledSubAgent` 已经编译好（可以是 `createUniverseAgent` 的结果）。

## task 工具

### 工具 Schema

```typescript
schema: z.object({
  description: z.string().describe("The task to execute with the selected agent"),
  subagent_type: z.string().describe("Name of the agent to use. Available: ..."),
})
```

### 执行流程

```typescript
async function taskTool(input, config) {
  const { description, subagent_type } = input;

  // 1. 校验子代理类型
  if (!(subagent_type in subagentGraphs)) {
    throw new Error(`Error: invoked agent of type ${subagent_type}...`);
  }

  // 2. 获取当前状态并过滤
  const currentState = getCurrentTaskInput<Record<string, unknown>>();
  const subagentState = filterStateForSubagent(currentState);
  // 用任务描述替换消息历史
  subagentState.messages = [new HumanMessage({ content: description })];

  // 3. 调用子代理
  const result = await subagent.invoke(subagentState, config);

  // 4. 返回结果
  return returnCommandWithStateUpdate(result, config.toolCall.id);
}
```

### 状态过滤

调用子代理时，从主 Agent 状态中排除以下字段：

```typescript
const EXCLUDED_STATE_KEYS = [
  "messages",          // 用任务描述替换
  "todos",             // 无 Reducer，不传递
  "structuredResponse", // 无 Reducer，不传递
  "skillsMetadata",    // 各 Agent 独立加载技能
  "memoryContents",    // 各 Agent 独立加载记忆
];
```

其他状态字段（如 `files`）会传递给子代理，使子代理能访问主 Agent 已有的文件。

### 结果处理

```typescript
function returnCommandWithStateUpdate(result, toolCallId) {
  const stateUpdate = filterStateForSubagent(result);

  // 结构化响应优先
  let content;
  if (result.structuredResponse != null) {
    content = JSON.stringify(result.structuredResponse);
  } else {
    // 取子代理最后一条消息
    const lastMessage = result.messages?.[result.messages.length - 1];
    content = lastMessage?.content || "Task completed";

    // 过滤无效 block 类型（tool_use、thinking 等）
    if (Array.isArray(content)) {
      content = content.filter(
        block => !["tool_use", "thinking", "redacted_thinking"].includes(block.type)
      );
    }
  }

  return new Command({
    update: {
      ...stateUpdate,  // 子代理产生的文件等状态更新回主 Agent
      messages: [new ToolMessage({ content, tool_call_id: toolCallId, name: "task" })],
    }
  });
}
```

**关键**：子代理执行过程中写入的文件（`files` 状态更新）会通过 `Command` 返回到主 Agent，实现跨子代理的文件共享。

## task 工具的系统提示

```typescript
export const TASK_SYSTEM_PROMPT = `
## \`task\` (subagent spawner)

你有一个 \`task\` 工具来启动短暂的子代理处理隔离任务...

何时使用 task 工具：
- 任务复杂且多步骤，可以完全独立委派
- 任务可以并行运行
- 任务需要专注的推理或大量 token/上下文

何时不用 task 工具：
- 需要看到中间步骤
- 任务简单（几个工具调用）
- 拆分不会减少 token 使用或复杂度

重要提示：
- 尽可能并行化！同时发起多个 task 调用
- 对于包含多个独立步骤的目标，使用 task 来隔离
`;
```

## 子代理并行执行

LLM 可以在一条消息中发起多个 `task` 工具调用（并行）：

```
LLM 输出：
  ToolCall: task(description="Research topic A", subagent_type="researcher")
  ToolCall: task(description="Research topic B", subagent_type="researcher")
  ToolCall: task(description="Research topic C", subagent_type="researcher")

LangGraph 并行执行：
  subagent_A.invoke(...)  ─┐
  subagent_B.invoke(...)   ├─ 并行
  subagent_C.invoke(...)  ─┘

结果合并（通过 fileDataReducer）：
  files: { ...A_files, ...B_files, ...C_files }
```
