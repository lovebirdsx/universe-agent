# 录像模式

**文件**：`src/recording.ts`

录像模式允许将 Agent 的完整执行过程（LLM 调用 + 工具执行）录制到磁盘，之后无需调用真实 LLM 即可完整回放。主要用于**测试确定性复现**和**调试分析**。

---

## 架构概览

```
createDeepAgent({ recording: { mode, path?, id? } })
    │
    ▼
resolveEffectiveMode()
    │
    ├─ 'record' ─────────────────────────────────── 'replay'
    │                                                  │
    ▼                                                 ▼
new Recorder()                               buildReplayModel()
createRecordingModel(model, recorder)          → fakeModel + 响应队列
createToolRecordingMiddleware()               loadToolResults()
    │                                          → Map<toolCallId, ToolMessage>
    │                                         createToolReplayMiddleware()
    ▼                                                  │
拦截 invoke/stream 调用                                 ▼
    │                                         工具调用直接返回录制结果
    ▼                                         LLM 调用返回预设响应
recorder.flush()
    ├── manifest.json
    ├── <agent>.recording.json
    └── transcript.md
```

## 配置

```typescript
interface RecordingConfig {
  mode: 'record' | 'replay' | 'auto';
  /** 录像目录基路径，默认 './.data/recordings' */
  path?: string;
  /** 录像 ID，默认为 process.argv[1] 相对于 cwd 的路径 */
  id?: string;
}
```

**使用示例**：

```typescript
const agent = createDeepAgent({
  model: 'anthropic:claude-sonnet-4-6',
  tools: [...],
  recording: { mode: 'auto' },
});
```

## 三种模式

### `record` — 强制录制

始终创建新录像，覆盖已有数据。

### `replay` — 强制回放

严格从已有录像文件回放，不调用真实 LLM 和工具。录像不存在时报错。

### `auto` — 智能选择（推荐）

`resolveEffectiveMode()` 自动判断：

| 条件                                        | 结果     |
| ------------------------------------------- | -------- |
| manifest.json 不存在                        | `record` |
| manifest 解析失败                           | `record` |
| status 为 `recording`（上次未完成）         | `record` |
| `recordingVersion` 与当前不匹配             | `record` |
| status 为 `completed` 或 `error` 且版本匹配 | `replay` |

## 录像文件结构

```
.data/recordings/<id>/
├── manifest.json              # 全局元数据 + 执行序列
├── main.recording.json        # 主 Agent 的模型/工具调用记录
├── <subagent>.recording.json  # 子 Agent 的记录（按名称区分）
└── transcript.md              # 人类可读的 Markdown 对话记录
```

### manifest.json

```typescript
interface ManifestData {
  viewVersion: number;           // 视图版本号（格式变更不影响回放）
  recordingVersion: number;      // 录像版本号（不兼容变更时递增）
  id: string;                    // 录像 ID
  createdAt: string;             // 创建时间
  completedAt?: string;          // 完成时间
  status: 'recording' | 'completed' | 'error';
  sequence: SequenceEntry[];     // 全局执行顺序
}

interface SequenceEntry {
  type?: 'model' | 'tool';      // 条目类型
  agent: string;                 // Agent 名称（'main' 或子 Agent 名）
  index: number;                 // 在该 Agent turns 数组中的索引
  toolCallId?: string;           // tool 条目专用：工具调用 ID
}
```

`sequence` 数组记录了所有 Agent 的执行顺序，是回放时重建调用链的关键。

### Agent Recording 文件

```typescript
interface AgentRecording {
  version: 2;
  agent: string;                         // Agent 名称
  tools?: SerializedToolDefinition[];    // OpenAI function 格式的工具定义
  turns: (ModelTurn | ToolTurn)[];       // 按时间顺序的调用记录
}

interface ModelTurn {
  type: 'model';
  index: number;
  request: StoredMessage[];       // 增量请求消息（首次为完整历史）
  requestTotalLength: number;     // 完整输入的消息总数
  response: StoredMessage;        // 模型响应
}

interface ToolTurn {
  type: 'tool';
  index: number;
  toolCallId: string;             // 对应 LLM tool_call 的 ID
  toolName?: string;              // 工具名称
  toolArgs?: Record<string, unknown>;  // 工具参数
  result: StoredMessage;          // 工具执行结果
}
```

### transcript.md

自动生成的人类可读 Markdown，按全局 `sequence` 顺序输出：
- 各 Agent 的工具声明列表
- 每次 LLM 调用的输入（增量消息）和输出
- 每次工具调用的参数和结果
- 支持多种 content block：text、image、tool_use、tool_result、thinking 等

## 版本控制

```typescript
const CURRENT_VIEW_VERSION = 1;       // 格式变更但不影响回放
const CURRENT_RECORDING_VERSION = 1;  // 不兼容变更，版本不匹配时需重录
```

- `viewVersion`：transcript 格式等展示层变更时递增
- `recordingVersion`：数据结构不兼容变更时递增，`auto` 模式下版本不匹配自动触发重录

## 核心实现

### Recorder 类

负责在录制模式下收集所有数据：

```typescript
class Recorder {
  responses: Map<string, StoredMessage[]>;       // 模型输出（按 agent 分组）
  toolResults: Map<string, StoredMessage[]>;     // 工具结果（按 agent 分组）
  sequence: SequenceEntry[];                     // 全局执行顺序
  requests: Map<string, StoredMessage[][]>;      // 输入增量（按 agent 分组）
  requestLengths: Map<string, number[]>;         // 完整上下文长度

  registerTools(agentName, tools): void          // 注册工具定义
  record(agentName, message, inputMessages?): void         // 记录模型输出
  recordToolResult(agentName, message, toolCallId, ...): void  // 记录工具结果
  flush(dirPath, id, status): void               // 写入磁盘
}
```

**增量请求追踪**：

| 调用次数 | 行为                                 |
| -------- | ------------------------------------ |
| 首次     | 记录完整输入消息历史                 |
| 后续     | 仅记录自上次以来的新增消息           |
| 长度减小 | 检测到新 thread/会话，重置为完整记录 |

这种设计大幅减少了录像文件的体积。

### createRecordingModel(model, recorder, agentName)

用 **Proxy** 包装模型，透明拦截调用链：

```typescript
function createRecordingModel<T extends BaseLanguageModel>(
  model: T,
  recorder: Recorder,
  agentName: string = 'main',
): T
```

**拦截的方法**：

| 方法         | 行为                              |
| ------------ | --------------------------------- |
| `invoke`     | 执行后将输出和输入记录到 Recorder |
| `bind`       | 返回的新模型也用 Proxy 包装       |
| `bindTools`  | 同上                              |
| `withConfig` | 同上                              |

采用 Proxy 而非继承的原因：LangChain 的 `bind`/`bindTools`/`withConfig` 会返回 `RunnableBinding` 等包装类型，继承难以统一处理。Proxy 可以透明拦截整个调用链。

### buildReplayModel(dirPath)

从录像文件构建 `fakeModel`，按全局 `sequence` 顺序填充响应队列：

```typescript
function buildReplayModel(dirPath: string): BaseLanguageModel
```

1. 加载 `manifest.json`，校验 `recordingVersion`
2. 按 agent 名称加载各 `.recording.json`
3. 按 `sequence` 中 `type === 'model'` 的顺序，将 `ModelTurn.response` 排入 `fakeModel` 队列
4. **覆写 `bindTools`**：使用 `withConfig({})` 代替原生实现

> **bindTools 修复**：`fakeModel` 的 `bindTools` 每次创建新实例并值拷贝 `_callIndex`，导致计数器重置为 0，回放永远返回第一条录像。覆写后所有 binding 共享同一个 model 实例和计数器。

### loadToolResults(dirPath)

从录像加载所有工具结果，返回 `Map<toolCallId, BaseMessage>`，用于回放时跳过真实工具执行。

## 工具中间件

### 录制中间件 — `createToolRecordingMiddleware`

通过 `wrapToolCall` 拦截所有工具调用：

```typescript
wrapToolCall: async (request, handler) => {
  const result = await handler(request);  // 执行真实工具
  // 将 ToolMessage 结果录入 Recorder
  recorder.recordToolResult(agentName, result, toolCallId, toolName, toolArgs);
  return result;
}
```

支持两种结果类型：
- **ToolMessage**：直接录制
- **Command**：从 `update.messages` 中提取 ToolMessage 录制

### 回放中间件 — `createToolReplayMiddleware`

通过 `wrapToolCall` 拦截，**跳过真实执行**，直接返回录制结果：

```typescript
wrapToolCall: async (request) => {
  const recorded = toolResults.get(toolCallId);
  if (recorded) return recorded;
  throw new Error(`No recorded tool result found for tool call ID: ${toolCallId}`);
}
```

## 与 createDeepAgent 的集成

### 录制模式流程

```typescript
// 1. 创建 Recorder
recorder = new Recorder();

// 2. 用 Proxy 包装模型
resolvedModel = createRecordingModel(resolvedModel, recorder, agentName);

// 3. 注入工具录制中间件（排在中间件栈首位）
recordingToolMiddleware = createToolRecordingMiddleware({ recorder, agentName });

// 4. 包装 invoke/stream，完成后 flush
agent.invoke = async (input, options) => {
  try {
    const result = await originalInvoke(input, options);
    recorder.flush(dir, id, 'completed');
    return result;
  } catch (error) {
    recorder.flush(dir, id, 'error');
    throw error;
  }
};
```

### 回放模式流程

```typescript
// 1. 构建 fakeModel 替换真实模型
resolvedModel = buildReplayModel(recordingDirPath);

// 2. 加载工具结果
const toolResults = loadToolResults(recordingDirPath);

// 3. 注入工具回放中间件
recordingToolMiddleware = createToolReplayMiddleware({ toolResults });
```

### Streaming 支持

录制模式下 `stream()` 也被包装，流结束后自动 flush：

```typescript
agent.stream = async (input, options) => {
  const iterator = await originalStream(input, options);
  return wrapAsyncIterableWithFlush(iterator, recorder, dir, id);
};
```

`wrapAsyncIterableWithFlush` 使用 `AsyncGenerator` 包装：
- 正常结束 → `flush('completed')`
- 抛出异常 → `flush('error')` → 重新抛出

## 子 Agent 录像

每个子 Agent 拥有独立的录制/回放追踪：

| 特性           | 行为                                                         |
| -------------- | ------------------------------------------------------------ |
| 工具录制中间件 | 每个子 Agent 创建独立实例，agentName 设为子 Agent 名称       |
| 模型 Proxy     | 子 Agent 的 model 单独用 `createRecordingModel()` 包装       |
| 录像文件       | 按 Agent 名称分文件存储（如 `researcher.recording.json`）    |
| 全局序列       | 所有 Agent 的调用按执行顺序记录在 `manifest.sequence` 中     |
| 回放中间件     | 所有子 Agent 共享同一个 `ToolReplayMiddleware`（按 ID 匹配） |

## 文件 I/O 安全

所有文件写入使用**原子写入**：先写 `.tmp` 文件，再 `rename`：

```typescript
function writeJsonAtomic(filePath: string, data: unknown): void {
  const tmpPath = filePath + '.tmp';
  fs.writeFileSync(tmpPath, JSON.stringify(data, null, 2), 'utf-8');
  fs.renameSync(tmpPath, filePath);
}
```

防止进程中途崩溃导致文件损坏。
