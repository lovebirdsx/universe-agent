## 构建配置（tsdown.config.ts）

```typescript
export default {
  entry: ["./src/index.ts"],    // 单一入口
  format: ["esm", "cjs"],       // 双格式输出
  dts: true,                    // 生成类型声明文件
  outDir: "dist",               // 输出目录
  external: [/^[^./]/],        // 所有 node_modules 标记为外部依赖（不打包）
}
```

**输出文件**（`dist/` 目录）：
```
dist/
├── index.js          # ESM 格式（现代 JS）
├── index.cjs         # CommonJS 格式（Node.js 兼容）
├── index.d.ts        # ESM 类型声明
├── index.d.cts       # CJS 类型声明
└── *.js.map          # Source Map（调试用）
```

## TypeScript 配置（tsconfig.json）

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "dist"
  },
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules", "dist"]
}
```

根 `tsconfig.json`（工作区级别）通常包含：
- `"moduleResolution": "bundler"` 或 `"NodeNext"`
- `"strict": true`
- `"target": "ES2022"` 或更高

## 测试配置（vitest.config.ts）

```typescript
export default {
  test: {
    environment: "node",
    timeout: 60000,           // 60s 超时（集成测试需要）
    coverage: {
      provider: "v8"
    },
    // 集成测试通过 --mode int 激活
  }
}
```

**测试命令**：

```bash
# 单元测试（快速）
npm test

# 集成测试（需要 API Key，慢）
npm run test:int

# 监视模式（开发时）
npm run test:watch

# 覆盖率
npm run test:coverage
```

## Zod 版本说明

项目使用 `zod/v4` 导入路径（注意不是 `zod`）：

```typescript
// fs.ts 中的写法
import { z } from "zod/v4";
```

这是因为 Zod v4 支持 `v4` 子路径导入，同时保持向后兼容。如果你的项目使用 Zod v3，需要注意这个差异。
