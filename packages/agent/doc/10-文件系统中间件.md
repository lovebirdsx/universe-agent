# 文件系统中间件

**文件**：`src/middleware/fs.ts`

文件系统中间件是 DeepAgents 的核心能力，为 Agent 提供文件系统操作能力，同时实现大结果驱逐机制防止上下文溢出。

---

## 提供的工具

| 工具名       | Schema                                                | 说明                         |
| ------------ | ----------------------------------------------------- | ---------------------------- |
| `ls`         | `{ path?: string }`                                   | 列出目录内容，默认路径 `/`   |
| `read_file`  | `{ file_path, offset?, limit? }`                      | 按行读取文件，默认读 100 行  |
| `write_file` | `{ file_path, content }`                              | 创建/覆盖文件                |
| `edit_file`  | `{ file_path, old_string, new_string, replace_all? }` | 精确字符串替换               |
| `glob`       | `{ pattern, path? }`                                  | 文件 Glob 模式匹配           |
| `grep`       | `{ pattern, path?, glob? }`                           | 文本内容搜索                 |
| `execute`    | `{ command }`                                         | Shell 命令（仅沙箱后端支持） |

## 状态 Schema

```typescript
const FilesystemStateSchema = new StateSchema({
  files: new ReducedValue(
    z.record(z.string(), FileDataSchema).default(() => ({})),
    {
      inputSchema: z.record(z.string(), FileDataSchema.nullable()).optional(),
      reducer: fileDataReducer,
    }
  )
});
```

`files` 字段使用 `ReducedValue`（而非默认的 `LastValue`），允许并发子代理的文件操作正确合并。

## 文件 Reducer

```typescript
function fileDataReducer(
  current: FilesRecord | undefined,
  update: FilesRecordUpdate | undefined,
): FilesRecord {
  if (update === undefined) return current || {};
  if (current === undefined) {
    // 过滤掉删除操作（null 值）
    const result: FilesRecord = {};
    for (const [key, value] of Object.entries(update)) {
      if (value !== null) result[key] = value;
    }
    return result;
  }
  // 合并：update 中 null 值表示删除
  const result = { ...current };
  for (const [key, value] of Object.entries(update)) {
    if (value === null) delete result[key];
    else result[key] = value;
  }
  return result;
}
```

**并发安全**：两个子代理同时写不同文件时，各自的更新会正确合并，不会互相覆盖。

## 大结果驱逐机制

### 工具结果驱逐（wrapToolCall）

当工具返回结果超过阈值（默认 20000 tokens ≈ 80KB）时：

```
工具执行 → 返回大结果
    │
    ▼
写入文件 /large_tool_results/{sanitized_tool_call_id}
    │
    ▼
替换结果为：
  "工具调用 {id} 结果已保存到 /large_tool_results/{id}
   预览（前5行 + ... + 后5行）：
   {content_sample}"
```

**排除驱逐的工具**：`ls`, `glob`, `grep`, `read_file`, `edit_file`, `write_file`

- `ls/glob/grep`：自带截断，输出不会很大
- `read_file`：截断会导致 Agent 重复调用，无意义
- `edit_file/write_file`：只返回确认消息，不会很大

### HumanMessage 驱逐（beforeAgent）

当用户消息超过阈值（默认 50000 tokens ≈ 200KB）时：

```typescript
// beforeAgent 钩子
async beforeAgent(state) {
  const last = state.messages[state.messages.length - 1];
  if (!HumanMessage.isInstance(last)) return;

  const contentStr = extractTextFromMessage(last);
  if (contentStr.length > threshold) {
    // 写入文件
    const filePath = `/conversation_history/${randomId}`;
    await resolvedBackend.write(filePath, contentStr);

    // 标记消息（additional_kwargs.lc_evicted_to）
    // 实际截断在 wrapModelCall 中处理
  }
}
```

截断后替换为：
```
"消息内容过大，已保存到文件 {filePath}，使用 read_file 工具读取完整内容。
 预览：{content_sample}"
```

## 内容预览生成

```typescript
function createContentPreview(
  contentStr: string,
  headLines = 5,
  tailLines = 5,
): string {
  const lines = contentStr.split("\n");

  if (lines.length <= headLines + tailLines) {
    // 小文件：显示全部
    return formatContentWithLineNumbers(lines, 1);
  }

  // 大文件：显示头部 + 截断标记 + 尾部
  const head = lines.slice(0, headLines);
  const tail = lines.slice(-tailLines);
  const truncated = lines.length - headLines - tailLines;

  return [
    formatContentWithLineNumbers(head, 1),
    `\n... [${truncated} lines truncated] ...\n`,
    formatContentWithLineNumbers(tail, lines.length - tailLines + 1),
  ].join("");
}
```

## read_file 的分页机制

```typescript
schema: z.object({
  file_path: z.string(),
  offset: z.coerce.number().optional().default(0),   // 起始行（0-indexed）
  limit:  z.coerce.number().optional().default(100), // 最大读取行数
})
```

返回带行号的格式（类似 `cat -n`）：

```
   1  import { createAgent } from "langchain";
   2  
   3  export function createDeepAgent(params) {
   4    ...
   5  }
```

对于超长行（> 5000 字符）会自动分割为续行（5.1、5.2 等格式）。

## 二进制文件支持

`read_file` 检测文件 MIME 类型，非文本文件以 Base64 返回：

```typescript
if (!isTextMimeType(mimeType)) {
  // 检查大小限制（10MB）
  if (sizeBytes > MAX_BINARY_READ_SIZE_BYTES) return error;

  // 按 MIME 类型返回不同 block 类型
  if (mimeType.startsWith("image/")) return [{ type: "image", mimeType, data: base64 }];
  if (mimeType.startsWith("audio/")) return [{ type: "audio", mimeType, data: base64 }];
  if (mimeType.startsWith("video/")) return [{ type: "video", mimeType, data: base64 }];
  return [{ type: "file", mimeType, data: base64 }];
}
```

## execute 工具的动态注册

`execute` 工具**总是注册**，但在模型调用时动态过滤：

```typescript
// wrapModelCall 中
const supportsExecution = isSandboxBackend(resolvedBackend);
let tools = request.tools;
if (!supportsExecution) {
  tools = tools.filter(t => t.name !== "execute");
}
```

同时系统提示也动态调整：
```typescript
if (supportsExecution) {
  filesystemPrompt = `${filesystemPrompt}\n\n${EXECUTION_SYSTEM_PROMPT}`;
}
```
