# 技能中间件

**文件**：`src/middleware/skills.ts`、`src/skills/loader.ts`

技能中间件实现了一种 Markdown 驱动的 Agent 能力扩展机制，类似于"插件商店"——将可组合的指令以文件形式加载到 Agent 的系统提示中。

---

## 技能规范（SKILL.md 格式）

技能文件是包含 YAML frontmatter 的 Markdown 文件：

```markdown
---
name: web-search
description: Use Tavily to search the web for current information
license: MIT
compatibility: Requires TAVILY_API_KEY environment variable
allowedTools:
  - search_web
metadata:
  version: "1.0.0"
  author: "team-name"
---

# Web Search Skill

This skill enables the agent to search the web using the Tavily API.

## Usage

Use the `search_web` tool with a search query...

## Best Practices

- Be specific in your queries
- Combine multiple searches for comprehensive research
```

## SkillMetadata 类型

```typescript
interface SkillMetadata {
  name: string;          // 技能名（1-64字符，小写字母/数字/连字符）
  description: string;   // 描述（1-1024字符）
  path: string;          // SKILL.md 文件路径
  license?: string | null;
  compatibility?: string | null;  // 环境要求（最长500字符）
  metadata?: Record<string, string>;  // 自定义键值对
  allowedTools?: string[];  // 推荐工具名（用于渐进式披露）
}
```

```typescript
export const MAX_SKILL_FILE_SIZE = 10 * 1024 * 1024;  // 10MB
```

## 技能加载机制

### 路径与搜索

技能从后端路径加载，支持多个源路径：

```typescript
createSkillsMiddleware({
  backend,
  sources: [
    "/skills/user/",     // 用户级技能
    "/skills/project/",  // 项目级技能
  ]
})
```

**覆盖规则**：相同名称的技能，后面的源会覆盖前面的（"最后一个获胜"）。这实现了分层配置：
```
/skills/user/          → 基础技能
/skills/project/       → 项目覆盖
```

### 目录结构约定

```
/skills/
├── web-search/
│   └── SKILL.md          # 技能入口文件
├── code-review/
│   └── SKILL.md
└── data-analysis/
    ├── SKILL.md
    └── examples/         # 技能可以有子目录（当作文档资源）
        └── example.py
```

## 渐进式披露（Progressive Disclosure）

技能内容**不会一次性全部注入**系统提示。中间件将技能元数据（名称 + 描述）注入，仅在 LLM 需要使用某个技能时才展示完整内容：

### beforeAgent 钩子

```typescript
async beforeAgent(state) {
  // 从后端加载所有技能的元数据
  const skills = await loadSkillsFromSources(backend, sources);

  return {
    skillsMetadata: skills.map(s => ({
      name: s.name,
      description: s.description,
    }))
  };
}
```

### wrapModelCall 钩子

```typescript
async wrapModelCall(request, handler) {
  const skillsMetadata = request.state.skillsMetadata;

  // 生成技能概览（只含名称和描述）
  const skillsOverview = skillsMetadata
    .map(s => `- **${s.name}**: ${s.description}`)
    .join("\n");

  // 注入到系统提示（概览）
  const systemMessage = request.systemMessage.concat(`
    ## Available Skills
    ${skillsOverview}

    To use a skill, reference it by name in your response.
    The full skill instructions will be loaded when needed.
  `);

  return handler({ ...request, systemMessage });
}
```

## 状态 Schema

```typescript
stateSchema: z.object({
  skillsMetadata: z.array(z.object({
    name: z.string(),
    description: z.string(),
  })).default([])
})
```

`skillsMetadata` 被排除在子代理状态传递之外（`EXCLUDED_STATE_KEYS`），每个 Agent 独立加载自己的技能。
