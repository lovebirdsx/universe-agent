# 错误处理

**文件**：`src/errors.ts`

---

## ConfigurationError

DeepAgents 目前只定义了一种错误类型：`ConfigurationError`，用于 Agent 构建阶段（`createDeepAgent` 调用时）的配置校验错误。

```typescript
// 错误代码枚举（扩展点）
type ConfigurationErrorCode = "TOOL_NAME_COLLISION";

class ConfigurationError extends Error {
  [CONFIG_ERROR_SYMBOL] = true as const;
  override readonly name: string = "ConfigurationError";
  readonly code: ConfigurationErrorCode;

  constructor(message: string, code: ConfigurationErrorCode) {
    super(message);
    Object.setPrototypeOf(this, ConfigurationError.prototype);
    this.code = code;
  }

  // 跨 realm instanceof 检查
  static isInstance(error: unknown): error is ConfigurationError {
    return (
      typeof error === "object" &&
      error !== null &&
      (error as Record<symbol, unknown>)[CONFIG_ERROR_SYMBOL] === true
    );
  }
}
```

| `TOOL_NAME_COLLISION` | 用户传入的工具名与内置工具重名 | `"Tool name(s) [ls, grep] conflict with built-in tools. Rename your custom tools."` |

## 跨 realm 问题与解决方案

**问题**：当库以 ESM 和 CJS 两种格式同时发布时，可能存在多个 `ConfigurationError` 类（每种格式各有一个），导致 `instanceof` 检查失败：

```typescript
// 可能失败
if (error instanceof ConfigurationError) { ... }  // ❌ 跨 realm 失败
```

**解决方案**：使用全局 Symbol 作为实例标记：

```typescript
const CONFIG_ERROR_SYMBOL = Symbol.for("configuration.error");
// Symbol.for 保证跨模块相同 key 返回同一个 Symbol
```

然后通过 `static isInstance()` 方法代替 `instanceof`：

```typescript
// 推荐写法
if (ConfigurationError.isInstance(error)) { ... }  // ✓ 跨 realm 安全
```

## SandboxError（后端错误）

**文件**：`src/backends/protocol.ts`

沙箱后端操作的错误类型，设计与 `ConfigurationError` 类似：

```typescript
type SandboxErrorCode =
  | "NOT_INITIALIZED"      // 未初始化
  | "ALREADY_INITIALIZED"  // 已初始化（不可重复）
  | "COMMAND_TIMEOUT"      // 命令超时
  | "COMMAND_FAILED"       // 命令执行失败
  | "FILE_OPERATION_FAILED"; // 文件操作失败

class SandboxError extends Error {
  [SANDBOX_ERROR_SYMBOL] = true as const;
  override readonly name = "SandboxError";

  constructor(
    message: string,
    public readonly code: string,
    public readonly cause?: Error,  // 链式错误
  ) {
    super(message);
    Object.setPrototypeOf(this, SandboxError.prototype);
  }

  static isInstance(error: unknown): error is SandboxError { ... }
}
```

**使用示例**：
```typescript
try {
  await sandbox.execute("some command");
} catch (error) {
  if (SandboxError.isInstance(error)) {
    switch (error.code) {
      case "NOT_INITIALIZED":
        await sandbox.initialize();
        break;
      case "COMMAND_TIMEOUT":
        console.error("Command took too long");
        break;
    }
  }
}
```

---

## 错误处理设计原则

1. **结构化错误码**：每种错误都有 `code` 字段，方便程序化处理
2. **人类可读消息**：`message` 字段直接说明问题和解决方向
3. **跨 realm 安全**：用 `Symbol.for` + 静态 `isInstance` 代替 `instanceof`
4. **错误链**：`SandboxError` 支持 `cause` 参数，保留原始错误上下文
5. **仅在构建阶段抛错**：运行时错误（如文件不存在）通过结果对象的 `error` 字段返回，而非抛出异常
