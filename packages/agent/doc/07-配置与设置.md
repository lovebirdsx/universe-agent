# 配置与设置

**文件**：`src/config.ts`

提供项目根目录检测和用户级配置目录管理，类似于 `.git` 根目录和 `~/.deepagents` 用户目录的管理。

---

## Settings 接口

```typescript
interface Settings {
  // 项目根目录（.git 所在目录，null 如果没有找到）
  projectRoot: string | null;

  // 用户级 deepagents 目录（~/.deepagents）
  userDeepagentsDir: string;

  // 是否在项目根目录中
  hasProject: boolean;

  // Agent 专用目录
  getAgentDir(agentName: string): string;
  ensureAgentDir(agentName: string): string;

  // 技能目录
  getUserSkillsDir(agentName: string): string;
  ensureProjectSkillsDir(): string | null;

  // AGENTS.md 文件路径
  getUserAgentsMarkdownPath(agentName: string): string;
  getProjectAgentsMarkdownPath(): string | null;
}
```

## 目录结构约定

```
# 用户级目录
~/.deepagents/
├── AGENTS.md                    # 用户全局记忆
└── agents/
    └── {agentName}/
        ├── AGENTS.md            # Agent 专用记忆
        └── skills/
            └── {skill-name}/
                └── SKILL.md     # 技能文件

# 项目级目录
{projectRoot}/
├── .deepagents/
│   ├── AGENTS.md                # 项目记忆
│   └── skills/
│       └── {skill-name}/
│           └── SKILL.md
└── .git/                        # 项目根标识
```

## findProjectRoot

```typescript
function findProjectRoot(startDir?: string): string | null
```

从给定目录（默认为 `process.cwd()`）向上搜索，直到找到包含 `.git` 目录的目录：

```typescript
// 简化实现逻辑
function findProjectRoot(startDir = process.cwd()): string | null {
  let current = startDir;
  while (true) {
    if (existsSync(join(current, ".git"))) return current;
    const parent = dirname(current);
    if (parent === current) return null;  // 到达文件系统根
    current = parent;
  }
}
```

## createSettings

```typescript
function createSettings(options?: SettingsOptions): Settings
```

`SettingsOptions` 可以指定：
- `projectRoot`：手动覆盖项目根目录
- `userHomeDir`：手动覆盖用户主目录（测试用）

---

## 使用场景

主要用于 CLI 工具和技能加载时的路径解析：

```typescript
import { createSettings, findProjectRoot } from "deepagents";

const settings = createSettings();

// 加载用户技能
const userSkillsDir = settings.getUserSkillsDir("my-agent");

// 加载项目记忆
const projectMemoryPath = settings.getProjectAgentsMarkdownPath();

// 检查是否在 Git 仓库中
if (settings.hasProject) {
  console.log("Project root:", settings.projectRoot);
}
```

## errors.ts — 配置错误

**文件**：`src/errors.ts`

```typescript
type ConfigurationErrorCode = "TOOL_NAME_COLLISION";

class ConfigurationError extends Error {
  readonly name = "ConfigurationError";
  readonly code: ConfigurationErrorCode;

  constructor(message: string, code: ConfigurationErrorCode) {
    super(message);
    this.code = code;
  }

  // 跨 realm 的 instanceof 检查（解决打包后类继承问题）
  static isInstance(error: unknown): error is ConfigurationError {
    return (
      typeof error === "object" &&
      error !== null &&
      (error as any)[CONFIG_ERROR_SYMBOL] === true
    );
  }
}
```

**设计亮点**：使用 `Symbol.for("configuration.error")` 作为实例标识，解决了 ESM/CJS 双格式打包后 `instanceof` 跨 realm 失效的问题。

export const filesValue = new ReducedValue(
