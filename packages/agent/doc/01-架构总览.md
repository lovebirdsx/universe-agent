# 架构总览

## 系统定位

DeepAgents 是一个**生产级 AI Agent 运行时框架**，建立在 LangGraph（状态机图执行引擎）之上，提供"开箱即用"的 Agent 能力：

- **文件系统工作记忆**：Agent 可以在会话内读写文件，跨步骤共享信息
- **子代理委派**：主代理可将子任务分发给专用子代理，支持并行执行
- **技能系统**：可动态加载 Markdown 格式的技能指令，实现渐进式能力扩展
- **自动历史压缩**：对话过长时自动摘要，防止 token 溢出
- **可插拔后端**：文件存储可以是内存状态、LangGraph Store、本地磁盘或沙箱

                           │

│  ⑥ patchToolCallsMiddleware – 修复悬挂工具调用                 │

│  ⑧ [customMiddleware]       – 用户自定义（可选）               │

│  ⑪ [humanInTheLoop]         – 人工审批中断（可选）              │
└────────────┬────────────────────────────────────────────────────┘
             │ 工具调用时路由到后端
┌────────────▼────────────────────────────────────────────────┐
│                       后端层（Backend）                      │
│                                                              │
│  StateBackend      – 存储在 LangGraph 状态（默认）           │

│  CompositeBackend  – 多后端路由组合                          │

## 数据流

### Agent 调用流程

```
用户调用 agent.invoke({ messages: [...] })
    │
    ▼
LangGraph 图引擎启动
    │
    ├─ beforeAgent 钩子（各中间件）
    │   ├─ 检查 HumanMessage 大小 → 超限则驱逐到文件系统
    │   └─ 加载记忆/技能到系统提示
    │
    ├─ wrapModelCall 钩子（各中间件）
    │   ├─ 追加系统提示（文件系统、子代理、技能、记忆等）
    │   ├─ 过滤工具列表（如 execute 需沙箱支持）
    │   └─ 应用 Anthropic 提示缓存标记
    │
    ├─ LLM 推理（生成回复 + 工具调用）
    │
    ├─ wrapToolCall 钩子（各中间件）
    │   ├─ 执行实际工具（文件读写、子代理调用等）
    │   └─ 工具结果超限 → 驱逐到文件系统并替换为引用
    │
    └─ 循环执行直到 LLM 不再调用工具（或触发中断）
            │
            ▼
       返回最终状态（messages + files + 自定义状态）
```

### 文件状态流

```
写入文件:
  write_file 工具 → 后端 write()
      ├─ StateBackend: 返回 filesUpdate → LangGraph 状态合并
      └─ FilesystemBackend: 直接写磁盘 → filesUpdate=null

读取文件:
  read_file 工具 → 后端 read() → 返回内容给 LLM

文件状态合并（并发安全）:
  fileDataReducer(current, update)
      ├─ update[key] = null → 删除文件
      └─ update[key] = FileData → 添加/覆盖文件
```

## 关键设计决策

### 1. 中间件顺序固定

中间件的执行顺序是硬编码的，不依赖用户排序，保证行为可预期：

```typescript
const middleware = [
  todoMiddleware,       // 最先 - 提供基础工具
  ...skillsMiddleware,  // 技能在文件系统之前加载
  fsMiddleware,         // 文件系统工具
  subagentMiddleware,   // 子代理（需要 fs 已注册）
  summarizationMiddleware,
  patchToolCallsMiddleware,
  ...asyncSubAgents,
  ...customMiddleware,  // 用户中间件最后
  ...cacheMiddleware,
  ...memoryMiddleware,
  ...hitlMiddleware,
];
```

### 2. 后端作为插件

所有文件操作通过 `BackendProtocolV2` 接口抽象，中间件代码不知道底层存储：

```typescript
interface BackendProtocolV2 {
  ls(path): Promise<LsResult>
  read(path, offset?, limit?): Promise<ReadResult>
  write(path, content): Promise<WriteResult>
  edit(path, old, new, replaceAll?): Promise<EditResult>
  glob(pattern, path?): Promise<GlobResult>
  grep(pattern, path?, glob?): Promise<GrepResult>
}
```

### 3. 状态通过 Reducer 合并

`files` 状态字段使用自定义 Reducer（`fileDataReducer`），而非 LangGraph 默认的"最后写入"语义，这样并发子代理的文件操作可以正确合并：

```typescript
// 并发子代理都写文件，各自的变更被合并而非覆盖
subagentA → files: { "/a.txt": ... }
subagentB → files: { "/b.txt": ... }
最终状态: files: { "/a.txt": ..., "/b.txt": ... }  ✓
```

### 4. 工具结果驱逐

当 LLM 工具返回结果超过 20000 tokens 时，自动将结果写入文件系统，并用包含路径和预览的摘要替换原内容，防止上下文溢出。

### 5. 自动 Anthropic 优化

系统自动检测是否使用 Anthropic 模型（通过 `isAnthropicModel()`），是则自动启用：
- `anthropicPromptCachingMiddleware`（提示缓存）
- `createCacheBreakpointMiddleware`（缓存断点控制）

## 与 LangGraph 的关系

DeepAgents 是 LangGraph 的**高层封装**：

```
DeepAgents.createDeepAgent()
    └─ langchain.createAgent()     ← LangChain 的 ReactAgent 工厂
           └─ @langchain/langgraph  ← 底层状态机图执行引擎
```

- **LangGraph** 提供：状态机图、checkpointing、状态持久化、`ReducedValue`、`Command`
- **LangChain** 提供：中间件机制、工具定义、模型抽象、消息类型
- **DeepAgents** 提供：完整的 Agent 运行时（文件系统 + 子代理 + 技能 + 记忆等）
